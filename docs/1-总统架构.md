## 1. 项目概述

### 1.1 项目背景

ToolBox 是一款面向桌面端的效率工具箱应用，主要面向开发者、设计师及日常办公用户。前期核心功能包括：

| 模块 | 描述 |
|------|------|
| 截图工具 | 全屏/区域/窗口截图，标注，延时截图 |
| 翻译工具 | 多引擎翻译（有道/DeepL/GPT），划词翻译 |
| 剪贴板管理 | 历史记录、分类、搜索、同步 |
| 插件系统 | 动态加载第三方插件，插件市场 |

### 1.2 设计目标

- **高性能**：UI 帧率 ≥ 60fps，截图响应 < 50ms
- **可扩展**：插件化架构，核心与功能解耦
- **跨平台**：Windows 10+、macOS 12+、Linux（Ubuntu 20.04+）
- **可维护**：清晰的分层架构，完善的日志与错误追踪

## 4. 架构设计

### 4.1 分层架构

```
┌─────────────────────────────────────────────┐
│              QML UI Layer                   │  ← 纯界面，不含业务逻辑
│     (main.qml / Pages / Components)        │
└─────────────────┬───────────────────────────┘
                  │  Q_PROPERTY / Q_INVOKABLE / signals
┌─────────────────▼───────────────────────────┐
│           ViewModel Layer                   │  ← QObject 子类，Qt 数据绑定
│  (ScreenshotVM / TranslateVM / ClipboardVM) │
└─────────────────┬───────────────────────────┘
                  │  C++ 函数调用 / std::function 回调
┌─────────────────▼───────────────────────────┐
│            Service Layer                    │  ← 纯业务逻辑，无 Qt UI 依赖
│   (ScreenshotService / TranslateService)    │
└─────────────────┬───────────────────────────┘
                  │
┌─────────────────▼───────────────────────────┐
│             Core Infrastructure             │  ← 跨模块基础设施
│    (Logger / EventBus / PluginLoader /      │
│     ConfigManager / ThreadPool)             │
└─────────────────────────────────────────────┘
```

### 4.2 关键设计模式

| 模式 | 应用场景 |
|------|----------|
| MVVM | QML ↔ ViewModel ↔ Service |
| 依赖注入 | AppContext 管理服务生命周期 |
| 观察者 / 信号槽 | ViewModel 向 QML 推送状态变更 |
| 策略模式 | 翻译引擎、截图驱动 |
| 插件模式 | 动态加载扩展功能 |
| Result<T, E> | 错误传播，禁用异常（可配置） |
| 命令模式 | 截图标注的撤销/重做 |

## 6. 接口规范（C++ / QML Bridge）

### 6.1 ViewModel 接口定义原则

**规则一：ViewModel 继承自 QObject，属性用 Q_PROPERTY**

```cpp
// src/viewmodels/TranslateViewModel.h

#pragma once
#include <QObject>
#include <QString>
#include <QtQml/qqmlregistration.h>
#include "services/translate/TranslateService.h"

class TranslateViewModel : public QObject {
    Q_OBJECT
    QML_ELEMENT                          // Qt 6: 自动注册到 QML

    // ── 属性声明 ──────────────────────────────────────────────────────────
    Q_PROPERTY(QString sourceText
               READ  sourceText
               WRITE setSourceText
               NOTIFY sourceTextChanged
               FINAL)

    Q_PROPERTY(QString resultText
               READ  resultText
               NOTIFY resultTextChanged
               FINAL)

    Q_PROPERTY(bool translating
               READ  isTranslating
               NOTIFY translatingChanged
               FINAL)

    Q_PROPERTY(QString errorMessage
               READ  errorMessage
               NOTIFY errorMessageChanged
               FINAL)

    Q_PROPERTY(QStringList supportedEngines
               READ  supportedEngines
               CONSTANT)

public:
    explicit TranslateViewModel(
        std::shared_ptr<TranslateService> service,
        QObject* parent = nullptr);

    // ── 属性 Getter ───────────────────────────────────────────────────────
    [[nodiscard]] QString     sourceText()      const noexcept;
    [[nodiscard]] QString     resultText()      const noexcept;
    [[nodiscard]] bool        isTranslating()   const noexcept;
    [[nodiscard]] QString     errorMessage()    const noexcept;
    [[nodiscard]] QStringList supportedEngines() const;

    // ── 属性 Setter ───────────────────────────────────────────────────────
    void setSourceText(const QString& text);

public slots:
    // ── QML 可调用方法（业务操作）────────────────────────────────────────
    Q_INVOKABLE void translate(const QString& text,
                               const QString& from = QStringLiteral("auto"),
                               const QString& to   = QStringLiteral("zh"));

    Q_INVOKABLE void switchEngine(const QString& engineId);
    Q_INVOKABLE void clearHistory();
    Q_INVOKABLE void copyResult();

signals:
    // ── 状态变更通知 ──────────────────────────────────────────────────────
    void sourceTextChanged();
    void resultTextChanged();
    void translatingChanged();
    void errorMessageChanged();

    // ── 业务事件（QML 监听）──────────────────────────────────────────────
    void translateSucceeded(const QString& result);
    void translateFailed(const QString& reason);
    void engineSwitched(const QString& engineId);

private:
    void onTranslateResult(const TranslateResult& result);
    void setTranslating(bool value);
    void setErrorMessage(const QString& msg);

    std::shared_ptr<TranslateService> m_service;
    QString     m_sourceText;
    QString     m_resultText;
    QString     m_errorMessage;
    bool        m_translating{false};
};
```

**规则二：禁止在 QML 直接操作 C++ 服务类**

```
QML → ViewModel → Service    ✅ 正确
QML → Service               ❌ 禁止（跳过 ViewModel）
QML → Q_INVOKABLE 直接 new 对象  ❌ 禁止
```


## 7. 数据结构规范

### 7.1 通用结果类型 `Result<T>`

```cpp
// src/core/error/Error.h

#pragma once
#include <variant>
#include <string>
#include <optional>
#include <stdexcept>

// ── 错误码枚举 ────────────────────────────────────────────────────────────
enum class ErrorCode : int32_t {
    // 通用
    Ok                  = 0,
    Unknown             = 1,
    InvalidArgument     = 2,
    Timeout             = 3,
    NetworkError        = 4,
    PermissionDenied    = 5,
    NotFound            = 6,
    IoError             = 7,

    // 截图
    ScreenCaptureError  = 1000,
    AnnotationError     = 1001,

    // 翻译
    TranslateApiError   = 2000,
    TranslateQuotaError = 2001,
    TranslateAuthError  = 2002,

    // 剪贴板
    ClipboardEmpty      = 3000,
    ClipboardReadError  = 3001,

    // 插件
    PluginLoadError     = 4000,
    PluginVersionMismatch = 4001,
    PluginDependencyMissing = 4002,
};

// ── 错误信息结构体 ────────────────────────────────────────────────────────
struct AppError {
    ErrorCode   code;
    std::string message;
    std::string details;    // 供调试，不展示给用户
    std::string location;   // __FILE__ + __LINE__（宏注入）

    static AppError make(ErrorCode code,
                         std::string message,
                         std::string details  = {},
                         std::string location = {}) {
        return {code, std::move(message), std::move(details), std::move(location)};
    }
};

// ── Result<T> ─────────────────────────────────────────────────────────────
template<typename T>
class Result {
public:
    // 成功构造
    static Result ok(T value) {
        Result r;
        r.m_data = std::move(value);
        return r;
    }

    // 失败构造
    static Result err(AppError error) {
        Result r;
        r.m_error = std::move(error);
        return r;
    }

    [[nodiscard]] bool isOk()  const noexcept { return m_data.has_value(); }
    [[nodiscard]] bool isErr() const noexcept { return m_error.has_value(); }

    // 获取值（调用前须检查 isOk()）
    [[nodiscard]] const T& value() const& {
        if (!m_data) throw std::logic_error("Result has no value");
        return *m_data;
    }
    [[nodiscard]] T&& value() && {
        if (!m_data) throw std::logic_error("Result has no value");
        return std::move(*m_data);
    }

    [[nodiscard]] const AppError& error() const& {
        if (!m_error) throw std::logic_error("Result has no error");
        return *m_error;
    }

    // map / and_then
    template<typename F>
    auto map(F&& f) const -> Result<std::invoke_result_t<F, const T&>> {
        using U = std::invoke_result_t<F, const T&>;
        if (isOk()) return Result<U>::ok(f(value()));
        return Result<U>::err(error());
    }

private:
    std::optional<T>        m_data;
    std::optional<AppError> m_error;
};

// ── 辅助宏 ────────────────────────────────────────────────────────────────
#define TB_MAKE_ERROR(code, msg) \
    AppError::make((code), (msg), {}, std::string(__FILE__) + ":" + std::to_string(__LINE__))

#define TB_RETURN_IF_ERR(expr)        \
    do {                              \
        auto&& _r = (expr);           \
        if (_r.isErr()) return _r;    \
    } while(0)
```

## 10. 错误处理规范

### 10.1 错误处理策略

```
原则一：业务逻辑层禁止使用 C++ 异常（性能 + 可预测性）
        → 使用 Result<T> 返回值传播错误

原则二：Qt 信号槽中的错误通过 signal 通知 ViewModel
        → 禁止在 Qt 事件处理中 throw

原则三：不可恢复错误（如 OOM）允许 std::terminate
        → 配合 signal handler 写 crash dump

原则四：所有外部 API 调用必须有错误检查
        → 不允许忽略 Result::isErr() 而直接 .value()
```

### 10.2 错误传播示例

```cpp
// Service 层
Result<TranslateResult> YoudaoEngine::translateSync(
    const TranslateRequest& req)
{
    if (req.text.isEmpty()) {
        return Result<TranslateResult>::err(
            TB_MAKE_ERROR(ErrorCode::InvalidArgument, "Text cannot be empty"));
    }

    auto httpResult = m_httpClient->post(buildUrl(req), buildBody(req));
    if (httpResult.isErr()) {
        TB_LOG_ERROR(LogModule::Translate,
            "HTTP request failed: {}", httpResult.error().message);
        return Result<TranslateResult>::err(httpResult.error());
    }

    auto parsed = parseResponse(httpResult.value());
    TB_RETURN_IF_ERR(parsed);

    return Result<TranslateResult>::ok(std::move(parsed).value());
}

// ViewModel 层：转换为 Qt 信号
void TranslateViewModel::onTranslateResult(const Result<TranslateResult>& result) {
    setTranslating(false);
    if (result.isOk()) {
        m_resultText = result.value().translatedText;
        emit resultTextChanged();
        emit translateSucceeded(m_resultText);
    } else {
        const auto& err = result.error();
        setErrorMessage(QString::fromStdString(err.message));
        emit translateFailed(QString::fromStdString(err.message));
        TB_LOG_ERROR(LogModule::Translate,
            "Translate failed [{}]: {}", static_cast<int>(err.code), err.message);
    }
}
```

```cpp
// 模板：键不存在时回落到结构体默认值，避免 crash
TranslateConfig fromJson(const QJsonObject& obj) {
    TranslateConfig cfg;  // 先用默认值填充
    cfg.defaultEngine   = obj["defaultEngine"].toString(cfg.defaultEngine);
    cfg.defaultFromLang = obj["defaultFromLang"].toString(cfg.defaultFromLang);
    cfg.defaultToLang   = obj["defaultToLang"].toString(cfg.defaultToLang);
    cfg.cacheMaxSize    = obj["cacheMaxSize"].toInt(cfg.cacheMaxSize);
    cfg.enableCache     = obj["enableCache"].toBool(cfg.enableCache);

    if (auto y = obj["youdao"].toObject(); !y.isEmpty()) {
        cfg.youdao.appId     = y["appId"].toString();
        cfg.youdao.appSecret = y["appSecret"].toString();
    }
    if (auto d = obj["deepl"].toObject(); !d.isEmpty()) {
        cfg.deepl.apiKey  = d["apiKey"].toString();
        cfg.deepl.freeApi = d["freeApi"].toBool(cfg.deepl.freeApi);
    }
    return cfg;
}

// ── 持久化读写（ConfigManager 内部）──────────────────────────────────────
void ConfigManager::save(const QString& path) const {
    QJsonObject root;
    root["translate"]  = toJson(m_config.translate);
    root["screenshot"] = toJson(m_config.screenshot);
    root["clipboard"]  = toJson(m_config.clipboard);

    QFile file(path);
    if (file.open(QIODevice::WriteOnly | QIODevice::Truncate))
        file.write(QJsonDocument(root).toJson(QJsonDocument::Indented));
}

Result<void> ConfigManager::load(const QString& path) {
    QFile file(path);
    if (!file.open(QIODevice::ReadOnly))
        return Result<void>::err(TB_MAKE_ERROR(ErrorCode::IoError,
            "Cannot open config file: " + path.toStdString()));

    QJsonParseError parseErr;
    auto doc = QJsonDocument::fromJson(file.readAll(), &parseErr);
    if (parseErr.error != QJsonParseError::NoError) {
        TB_LOG_ERROR(LogModule::App, "Config parse failed: {}",
            parseErr.errorString().toStdString());
        return Result<void>::err(TB_MAKE_ERROR(ErrorCode::InvalidArgument,
            parseErr.errorString().toStdString()));
    }

    auto root = doc.object();
    if (root.contains("translate"))
        m_config.translate = fromJson(root["translate"].toObject());
    if (root.contains("screenshot"))
        m_config.screenshot = fromJson(root["screenshot"].toObject());
    if (root.contains("clipboard"))
        m_config.clipboard  = fromJson(root["clipboard"].toObject());

    return Result<void>::ok({});
}
```

## 12. 多线程规范

### 12.1 线程模型

```
主线程 (Qt GUI Thread):
  - 所有 QML/UI 操作
  - ViewModel 的 slot/signal
  - 严禁阻塞 > 16ms（否则掉帧）

网络线程 (QNetworkAccessManager 线程池):
  - 所有 HTTP 请求（翻译 API 等）

截图线程 (专用 WorkerThread):
  - 屏幕捕获（避免阻塞主线程）
  - 图像压缩/处理

数据库线程 (专用 WorkerThread):
  - SQLite 读写（剪贴板历史持久化）

插件线程 (ThreadPool):
  - 插件的后台计算任务
```

### 12.2 线程安全规范

```cpp
// 规则一：跨线程通信只用 Qt 的队列信号槽（QueuedConnection）
// 禁止直接跨线程调用 QObject 的方法

// 规则二：共享数据用 std::mutex 或 QReadWriteLock
class ClipboardStorage {
    mutable QReadWriteLock m_lock;

    QList<ClipboardItem> m_items;

public:
    QList<ClipboardItem> items() const {
        QReadLocker locker(&m_lock);
        return m_items;  // 返回副本
    }

    void insert(ClipboardItem item) {
        QWriteLocker locker(&m_lock);
        m_items.prepend(std::move(item));
    }
};

// 规则三：从非 GUI 线程更新 UI，必须用 QMetaObject::invokeMethod
```

```cpp
// 函数名：camelCase          void captureRegion(QRect region);
// 变量名：camelCase          QString savePath;
// 成员变量：m_ 前缀          QString m_savePath;
// 常量/枚举：PascalCase      enum class CaptureMode { FullScreen };
// 宏：TB_ 前缀 + 全大写      #define TB_LOG_INFO(...)
// 命名空间：小写             namespace toolbox::translate {}

// ── 现代 C++ 要求 ─────────────────────────────────────────────────────────
// 1. 优先使用 auto（类型明确时），避免冗余类型声明
// 2. 使用 [[nodiscard]] 标注所有返回 Result<T> 的函数
// 3. 使用 std::span、std::string_view 替代原始指针参数
// 4. 智能指针：shared_ptr（共享所有权）, unique_ptr（独占），禁止裸 new/delete
// 5. 范围 for 循环代替索引循环（无需索引时）
// 6. 结构化绑定（auto& [key, value] = map.at(...)）
// 7. 移动语义：大对象参数用值 + std::move
// 8. constexpr / consteval 用于编译期常量

// ── 禁止项 ────────────────────────────────────────────────────────────────
// ❌ 禁止 using namespace std; （在头文件中）
// ❌ 禁止 C 风格转型 (Type)expr → 用 static_cast / qobject_cast
// ❌ 禁止忽略 [[nodiscard]] 返回值
// ❌ 禁止在构造函数中进行可能失败的操作 → 用工厂方法 + Result<T>
// ❌ 禁止 Qt4 风格的 SIGNAL()/SLOT() 宏 → 用函数指针形式
```

### 20.1 AppContext（依赖容器）

```cpp
// src/app/AppContext.h

#pragma once
#include <memory>
#include <typeindex>
#include <unordered_map>
#include <stdexcept>

/**
 * @brief 应用全局依赖容器（轻量级 IoC）
 *
 * 所有 Service 在 Application::init() 中注册，
 * ViewModel 通过 AppContext 获取所需 Service。
 *
 * 线程安全：只读访问（注册阶段结束后），可安全并发读取。
 */
class AppContext {
public:
    static AppContext& instance();

    template<typename T>
    void registerService(std::shared_ptr<T> service) {
        m_services[std::type_index(typeid(T))] =
            std::static_pointer_cast<void>(service);
    }

    template<typename T>
    [[nodiscard]] std::shared_ptr<T> resolve() const {
        auto it = m_services.find(std::type_index(typeid(T)));
        if (it == m_services.end()) {
            throw std::runtime_error(
                std::string("Service not registered: ") + typeid(T).name());
        }
        return std::static_pointer_cast<T>(it->second);
    }

private:
    AppContext() = default;
    std::unordered_map<std::type_index, std::shared_ptr<void>> m_services;
};

// 便捷宏
#define TB_RESOLVE(T) AppContext::instance().resolve<T>()
```

### 20.2 EventBus（全局事件总线）

```cpp
// src/core/event/EventBus.h

#pragma once
#include <functional>
#include <unordered_map>
#include <vector>
#include <typeindex>
#include <mutex>
#include <QObject>

// 所有事件类型继承此基类（空基类，仅用于类型约束）
struct AppEvent {};

struct ScreenshotTakenEvent : AppEvent {
    uint64_t screenshotId;
};

struct ClipboardChangedEvent : AppEvent {
    uint64_t newItemId;
};

struct TranslateCompletedEvent : AppEvent {
    QString sourceText;
    QString resultText;
    QString engineId;
};

struct PluginLoadedEvent : AppEvent {
    QString pluginId;
};

/**
 * @brief 跨模块事件总线（线程安全）
 *
 * 发布者：调用 publish<EventType>(event)
 * 订阅者：调用 subscribe<EventType>(handler)
 *
 * 注意：handler 在发布者线程调用，若需在 Qt 主线程处理，
 *       handler 内部须使用 QMetaObject::invokeMethod。
 */
class EventBus {
public:
    static EventBus& instance();

    using HandlerId = uint64_t;

    template<typename E>
    HandlerId subscribe(std::function<void(const E&)> handler) {
        static_assert(std::is_base_of_v<AppEvent, E>);
        std::lock_guard lock(m_mutex);
        auto id = ++m_nextId;
        m_handlers[std::type_index(typeid(E))].push_back({id,
            [h = std::move(handler)](const AppEvent& e) {
                h(static_cast<const E&>(e));
            }
        });
        return id;
    }

    void unsubscribe(HandlerId id);

    template<typename E>
    void publish(const E& event) {
        static_assert(std::is_base_of_v<AppEvent, E>);
        std::lock_guard lock(m_mutex);
        auto it = m_handlers.find(std::type_index(typeid(E)));
        if (it != m_handlers.end()) {
            for (auto& [hid, fn] : it->second) {
                fn(event);
            }
        }
    }

private:
    EventBus() = default;

    struct HandlerEntry {
        HandlerId id;
        std::function<void(const AppEvent&)> fn;
    };

    std::mutex m_mutex;
    std::unordered_map<std::type_index, std::vector<HandlerEntry>> m_handlers;
    HandlerId m_nextId{0};
};
```
