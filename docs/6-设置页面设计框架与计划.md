## 1. 概述

### 1.1 设置页面的战略意义

设置页面是 ToolBox 应用的**配置中枢**，承载以下职责：

- **用户偏好管理**：主题、语言、启动行为、快捷键等
- **功能模块配置**：截图、翻译、剪贴板各模块的参数调整
- **高级选项**：日志级别、缓存策略、性能优化、数据导出
- **系统信息**：版本、许可证、关于、帮助链接
- **插件管理**：第三方插件的启用/禁用、配置

### 1.2 设计约束

| 约束项 | 要求 | 说明 |
|--------|------|------|
| 设计风格 | Soft UI | 遵循《QML 开发主题规范》v1.0.0 |
| 架构模式 | MVVM | QML ↔ ViewModel ↔ Service ↔ Core |
| 错误处理 | Result<T> | 禁用异常，采用返回值模式 |
| 多线程 | 分离 | 配置读写在专用线程，UI 线程不阻塞 |
| 可扩展性 | 插件支持 | 插件可动态注册自己的设置项 |
| 性能 | 快速响应 | 设置项加载 < 500ms，应用 < 100ms |

## 2. 设计目标与原则

### 2.1 核心目标

1. **易用性**：用户能快速找到并修改所需设置
2. **可发现性**：设置项组织清晰，支持搜索
3. **安全性**：敏感设置（API Key、密码）加密存储
4. **可扩展性**：插件可无缝添加自己的设置项
5. **一致性**：所有设置项遵循统一的 UI 规范

### 2.2 设计原则

#### 原则一：分层组织，逐级深入

```
一级：分类导航（通用、截图、翻译、剪贴板、高级、关于）
  ↓
二级：功能分组（如"通用"下有"主题"、"语言"、"启动"等）
  ↓
三级：具体设置项（如"主题"下有"浅色"、"暗黑"、"跟随系统"）
```

#### 原则二：Soft UI 风格贯穿始终

- 所有设置项容器使用 **12-16px 圆角**
- 分组间使用 **柔和阴影**（Shadow-S 或 Shadow-M）表达层级
- 禁止硬边框，采用 **背景色 + 阴影** 区分
- 充足白空间：分组间距 **≥ 24px**，项目间距 **≥ 16px**

#### 原则三：实时反馈，即时预览

- 主题切换立即生效（无需重启）
- 语言切换立即生效
- 其他设置项支持"应用"按钮，点击后生效
- 敏感操作（如清空缓存）需要确认对话框

#### 原则四：渐进式披露

- 常用设置在"通用"分类中
- 高级选项放在"高级"分类中
- 默认隐藏不常用的选项，通过"更多"按钮展开

## 3. 页面架构设计

### 3.1 整体布局

采用 **双栏布局**（推荐）或 **标签页布局**（备选）：

#### 方案 A：双栏布局（推荐）

```
┌─────────────────────────────────────────────────────────────┐
│  ToolBox 设置                                          [×]   │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  ┌──────────────┐  ┌──────────────────────────────────────┐ │
│  │   分类导航   │  │         设置内容区                    │ │
│  │              │  │                                      │ │
│  │ ▸ 通用       │  │  通用设置                            │ │
│  │ ▸ 截图       │  │  ─────────────────────────────────  │ │
│  │ ▸ 翻译       │  │                                      │ │
│  │ ▸ 剪贴板     │  │  主题                                │ │
│  │ ▸ 高级       │  │  ○ 浅色  ○ 暗黑  ○ 跟随系统        │ │
│  │ ▸ 关于       │  │                                      │ │
│  │              │  │  语言                                │ │
│  │              │  │  [下拉选择器: 简体中文 ▼]           │ │
│  │              │  │                                      │ │
│  │              │  │  启动行为                            │ │
│  │              │  │  ☑ 开机自启                         │ │
│  │              │  │  ☑ 启动时恢复窗口位置               │ │
│  │              │  │                                      │ │
│  │              │  │  ─────────────────────────────────  │ │
│  │              │  │  [应用]  [重置]  [取消]             │ │
│  └──────────────┘  └──────────────────────────────────────┘ │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

**优点：**
- 导航和内容同时可见，上下文清晰
- 适合大屏幕（≥1200px）
- 易于实现搜索功能

**尺寸建议：**
- 左侧导航栏：240-280px
- 右侧内容区：自适应
- 最小窗口宽度：1000px

#### 方案 B：标签页布局（备选）

```
┌─────────────────────────────────────────────────────────────┐
│  ToolBox 设置                                          [×]   │
├─────────────────────────────────────────────────────────────┤
│  [通用] [截图] [翻译] [剪贴板] [高级] [关于]               │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  主题                                                        │
│  ○ 浅色  ○ 暗黑  ○ 跟随系统                                │
│                                                               │
│  语言                                                        │
│  [下拉选择器: 简体中文 ▼]                                   │
│                                                               │
│  启动行为                                                    │
│  ☑ 开机自启                                                 │
│  ☑ 启动时恢复窗口位置                                       │
│                                                               │
│  ─────────────────────────────────────────────────────────  │
│  [应用]  [重置]  [取消]                                     │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

**优点：**
- 适合小屏幕（< 1200px）
- 实现简单
- 移动端友好

**缺点：**
- 导航和内容不同时可见
- 标签页过多时难以管理

**推荐选择：方案 A（双栏布局）**，并在小屏幕时自动切换为方案 B。

### 3.2 响应式设计

```qml
// 伪代码：根据窗口宽度自动切换布局
SettingsPage {
    width: parent.width
    
    // 大屏幕（≥1000px）：双栏布局
    // 小屏幕（< 1000px）：标签页布局
    
    readonly property bool isWideScreen: width >= 1000
    
    RowLayout {
        visible: isWideScreen
        // 左侧导航 + 右侧内容
    }
    
    ColumnLayout {
        visible: !isWideScreen
        // 标签页 + 内容
    }
}
```

## 4. 信息架构（IA）

### 4.1 分类体系

设置页面分为 **6 个一级分类**，每个分类下有若干二级分组：

#### 分类 1：通用（General）

| 分组 | 设置项 | 类型 | 说明 |
|------|--------|------|------|
| 主题 | 主题模式 | 单选 | 浅色/暗黑/跟随系统 |
| 主题 | 主题色 | 选择 | 预设色彩方案 |
| 语言 | 界面语言 | 下拉 | 简体中文/English/日本語 |
| 语言 | 自动检测系统语言 | 开关 | 启用后自动跟随系统 |
| 启动 | 开机自启 | 开关 | Windows/macOS/Linux |
| 启动 | 启动时恢复窗口 | 开关 | 恢复上次关闭时的窗口位置/大小 |
| 启动 | 启动时显示主窗口 | 开关 | 否则启动到系统托盘 |
| 快捷键 | 全局快捷键设置 | 按钮 | 打开快捷键配置对话框 |

#### 分类 2：截图（Screenshot）

| 分组 | 设置项 | 类型 | 说明 |
|------|--------|------|------|
| 基础 | 截图保存路径 | 文件选择 | 默认 ~/Pictures/ToolBox |
| 基础 | 截图格式 | 下拉 | PNG/JPEG/WebP |
| 基础 | 截图质量 | 滑块 | 1-100，仅对 JPEG/WebP 有效 |
| 标注 | 默认笔刷颜色 | 颜色选择 | 标注工具的默认颜色 |
| 标注 | 默认笔刷宽度 | 滑块 | 1-20px |

| 延迟 | 延迟截图时间 | 输入框 | 秒数，0-60 |
| 高级 | 启用 OCR | 开关 | 需要 OCR 插件 |
| 高级 | 启用二维码识别 | 开关 | 需要 QR Code 插件 |

#### 分类 3：翻译（Translation）

## 5. UI 组件规范

### 5.1 设置项容器（SettingItem）

所有设置项都包装在统一的容器中，遵循 Soft UI 风格：

```qml
// src/ui/components/SettingItem.qml

Rectangle {
    id: root
    
    width: parent.width
    height: 64  // 基础高度，可根据内容调整
    
    // Soft UI 风格
    color: Theme.colorBackground
    radius: 12
    
    // 柔和阴影
    layer.enabled: true
    layer.effect: DropShadow {
        color: Theme.shadowColor
        opacity: 0.08
        blur: 8
        yOffset: 2
    }
    
    // 内容布局
    RowLayout {
        anchors.fill: parent
        anchors.margins: 16
        spacing: 16
        
        // 左侧：标签和描述
        ColumnLayout {
            Layout.fillWidth: true
            spacing: 4
            
            Text {
                text: root.label
                font: Theme.fontBody1
                color: Theme.colorText
            }
            
            Text {
                text: root.description
                font: Theme.fontCaption
                color: Theme.colorTextSecondary
                visible: root.description !== ""
            }
        }
        
        // 右侧：控件（开关、输入框、下拉等）
        Loader {
            sourceComponent: root.controlComponent
        }
    }
    
    // 属性
    property string label: ""
    property string description: ""
    property Component controlComponent
    
    // 禁用状态
    property bool enabled: true
    opacity: enabled ? 1.0 : 0.5
}
```

**使用示例：**

```qml
SettingItem {
    label: "开机自启"
    description: "应用启动时自动运行"
    
    controlComponent: Switch {
        checked: settingsVM.autoStartEnabled
        onToggled: settingsVM.setAutoStart(checked)
    }
}
```

```
   ┌────▼────┐        ┌────▼────┐       ┌────▼────┐
   │ 内置设置 │        │ 插件 A  │       │ 插件 B  │
```

```cpp
class SettingsRegistry {
public:
    // 注册设置项
    Result<void> registerSetting(
        const QString& pluginId,
        const SettingItem& item
    );
    
    // 注销设置项
    Result<void> unregisterSetting(
        const QString& pluginId,
        const QString& itemId
    );

    QList<SettingItem> getSettingsByCategory(
        const QString& category
    ) const;
    
private:
    QMap<QString, QList<SettingItem>> m_settings;  // pluginId -> items
};

// 设置项数据结构
struct SettingItem {
    QString id;                    // 唯一标识
    QString pluginId;              // 所属插件
    QString category;              // 分类（通用/截图/翻译等）
    QString group;                 // 分组
    QString label;                 // 显示标签
    QString description;           // 描述
    QString type;                  // 类型（switch/input/select/color等）

    QVariant currentValue;         // 当前值
    QMap<QString, QVariant> options;  // 额外选项（如下拉菜单的选项列表）
    bool requiresRestart = false;  // 是否需要重启应用
    bool isRealtime = false;       // 是否实时预览
```

### 8.2 实现细节

```cpp
// src/viewmodels/SettingsViewModel.cpp

SettingsViewModel::SettingsViewModel(
    ConfigManager* configManager,
    SettingsRegistry* registry,
    QObject* parent
) : QObject(parent),
    m_configManager(configManager),
    m_registry(registry),
    m_currentCategory("通用")
{
    connect(m_configManager, &ConfigManager::configChanged,
            this, &SettingsViewModel::onConfigChanged);
}

void SettingsViewModel::loadSettings(const QString& category) {
    m_currentCategory = category;
    loadSettingsFromRegistry();
    emit currentCategoryChanged();
    emit currentSettingsChanged();
}

void SettingsViewModel::updateSetting(
    const QString& itemId,
    const QVariant& value
) {
    m_pendingChanges[itemId] = value;
    emit unsavedChangesChanged();
}

void SettingsViewModel::applySettings() {
    // 验证数据
    auto validateResult = validateSettings();
    if (!validateResult.isOk()) {
        emit errorOccurred(validateResult.error().message());
        return;
    }
    
    // 应用更改
    auto applyResult = applyPendingChanges();
    if (!applyResult.isOk()) {
        emit errorOccurred(applyResult.error().message());
        return;
    }
    
    m_pendingChanges.clear();
    emit unsavedChangesChanged();
    emit settingsApplied();
    emit successMessage("设置已保存");
}

void SettingsViewModel::resetToDefaults() {
    m_configManager->resetToDefaults();
    m_pendingChanges.clear();
    loadSettingsFromRegistry();
    emit unsavedChangesChanged();
    emit settingsReset();
    emit successMessage("已恢复默认设置");
}

void SettingsViewModel::filterSettings(const QString& searchText) {
    m_searchText = searchText;
    
    // 搜索逻辑
    QList<QObject*> filtered;
    for (auto setting : m_currentSettings) {
        // 检查标签、描述等是否包含搜索文本
        if (setting->property("label").toString().contains(searchText, Qt::CaseInsensitive) ||
            setting->property("description").toString().contains(searchText, Qt::CaseInsensitive)) {
            filtered.append(setting);
        }
    }
    
    emit filteredSettingsChanged();
}
```

### 11.2 SettingsViewModel 实现示例

```cpp
// src/viewmodels/SettingsViewModel.cpp

#include "SettingsViewModel.h"
#include "../core/config/ConfigManager.h"
#include "../core/config/SettingsRegistry.h"

SettingsViewModel::SettingsViewModel(
    ConfigManager* configManager,
    SettingsRegistry* registry,
    QObject* parent
) : QObject(parent),
    m_configManager(configManager),
    m_registry(registry),
    m_currentCategory("通用")
{
    Q_ASSERT(m_configManager);
    Q_ASSERT(m_registry);
    
    connect(m_configManager, &ConfigManager::configChanged,
            this, &SettingsViewModel::onConfigChanged);
    
    loadSettingsFromRegistry();
}

void SettingsViewModel::loadSettings(const QString& category) {
    m_currentCategory = category;
    loadSettingsFromRegistry();
    emit currentCategoryChanged();
    emit currentSettingsChanged();
}

void SettingsViewModel::loadSettingsFromRegistry() {
    m_currentSettings.clear();
    
    auto settings = m_registry->getSettingsByCategory(m_currentCategory);
    
    for (const auto& setting : settings) {
        // 创建 QObject 包装器
        auto settingObj = new QObject(this);
        settingObj->setProperty("id", setting.id);
        settingObj->setProperty("label", setting.label);
        settingObj->setProperty("description", setting.description);
        settingObj->setProperty("type", setting.type);
        settingObj->setProperty("currentValue", setting.currentValue);
        settingObj->setProperty("options", QVariant::fromValue(setting.options));
        
        m_currentSettings.append(settingObj);
    }
}

void SettingsViewModel::updateSetting(
    const QString& itemId,
    const QVariant& value
) {
    m_pendingChanges[itemId] = value;
    emit unsavedChangesChanged();
}

void SettingsViewModel::applySettings() {
    // 验证数据
    auto validateResult = validateSettings();
    if (!validateResult.isOk()) {
        emit errorOccurred(validateResult.error().message());
        return;
    }
    
    // 应用更改
    for (auto it = m_pendingChanges.begin(); it != m_pendingChanges.end(); ++it) {
        auto saveResult = m_configManager->setValue(it.key(), it.value());
        if (!saveResult.isOk()) {
            emit errorOccurred(saveResult.error().message());
            return;
        }
    }
    
    // 保存到文件
    auto saveFileResult = m_configManager->save();
    if (!saveFileResult.isOk()) {
        emit errorOccurred(saveFileResult.error().message());
        return;
    }
    
    m_pendingChanges.clear();
    emit unsavedChangesChanged();
    emit settingsApplied();
}

void SettingsViewModel::resetToDefaults() {
    auto resetResult = m_configManager->resetToDefaults();
    if (!resetResult.isOk()) {
        emit errorOccurred(resetResult.error().message());
        return;
    }
    
    m_pendingChanges.clear();
    loadSettingsFromRegistry();
    emit unsavedChangesChanged();
    emit settingsReset();
}

Result<void> SettingsViewModel::validateSettings() {
    // 验证逻辑
    for (auto it = m_pendingChanges.begin(); it != m_pendingChanges.end(); ++it) {
        // 例如：验证数字范围
        if (it.key() == "cache_size_mb") {
            int size = it.value().toInt();
            if (size < 10 || size > 1000) {
                return Result<void>::error(
                    Error(ErrorCode::InvalidArgument, "缓存大小必须在 10-1000 MB 之间")
                );
            }
        }
    }
    
    return Result<void>::ok();
}
```
