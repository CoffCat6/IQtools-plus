## 1. 截图功能的特殊性

截图功能与翻译/AI/剪贴板完全不同，它是项目中**最复杂的模块**：

| 维度 | 翻译/AI | 截图 |
|------|---------|------|
| 平台依赖 | 无（纯 HTTP） | **重度依赖**（Win32/macOS/Linux 各不同） |
| UI 复杂度 | 普通页面 | **全屏覆盖层 + 实时交互** |
| 线程模型 | 主线程即可 | **专用截图线程 + 文件 I/O 线程** |
| 性能要求 | 宽松（200ms OK） | **苛刻（< 80ms 从热键到覆盖层）** |
| 数据大小 | KB 级文本 | **MB 级图像** |
| 交互模式 | 填表单、点按钮 | **拖拽、实时绘图、键盘快捷键** |

### 核心挑战

```
挑战 1：如何捕获屏幕内容？（平台 API）
  - Windows: DXGI OutputDuplication / BitBlt
  - macOS: CGDisplayCreateImage / CGWindowListCreateImage
  - Linux: XShmGetImage / XGetImage / Wayland wlr-screencopy

挑战 2：如何实现区域选择？（全屏覆盖层）
  - 创建一个无边框、全屏、置顶、透明的窗口
  - 在窗口上显示截图作为背景（半暗化）
  - 用户拖拽选择矩形区域
  - 实时显示选区尺寸

挑战 3：如何实现标注？（Canvas 绘制 + 命令模式）
  - 在 QML Canvas 上用 QPainter 绘制各种形状
  - 支持撤销/重做（Command Pattern）
  - 支持选中、移动、修改已有标注

挑战 4：如何保证性能？（线程模型）
  - 截图操作不能阻塞 UI 线程
  - 文件保存不能阻塞 UI 线程
  - 标注渲染要 60fps
```

## 2. 推荐实现路线图

**不要试图一次做完所有功能。** 按以下 5 个 Phase 逐步推进，每个 Phase 结束时都有一个可用的产物。

```
Phase 1（1-2 天）  → 能截图、能保存、能复制         ← 最小可用
Phase 2（3-5 天）  → 全屏覆盖层 + 区域选择           ← 核心体验
Phase 3（5-7 天）  → 标注引擎（矩形/箭头/文字）       ← 核心功能
Phase 4（3-5 天）  → 平台原生截图替换 Qt 原生         ← 性能优化
Phase 5（持续）    → 窗口截图/延时/历史/贴图/快捷键     ← 高级功能
```

```
Phase 1 ──────────────────────────────────────────────────────────────
  用 QScreen::grabWindow() 实现最简截图
  输出：点击按钮 → 截取全屏 → 保存到文件 / 复制到剪贴板
  
Phase 2 ──────────────────────────────────────────────────────────────
  实现 ScreenshotOverlay.qml 全屏覆盖层
  输出：热键 → 全屏暗化 → 拖拽选区 → 裁剪保存

Phase 3 ──────────────────────────────────────────────────────────────
  实现 AnnotationEngine + CommandHistory + ShapeRenderer
  输出：截图后进入标注模式 → 画矩形/箭头/文字 → 撤销/重做 → 保存

Phase 4 ──────────────────────────────────────────────────────────────
  用 DXGI/CGImage/XShm 替换 QScreen::grabWindow()
  输出：截图速度从 ~100ms 降到 ~15ms，支持多显示器

Phase 5 ──────────────────────────────────────────────────────────────
  窗口截图、延时截图、截图历史、贴图、全局热键
  输出：完整的截图工具
```

## 3. Phase 1：Qt 原生截图 + 最小可用工具

### 3.1 为什么从 Qt 原生开始

```
QScreen::grabWindow(0)  ← Qt 内置，零依赖，三平台通用
  - Windows: 内部用 BitBlt（GDI），~30ms@1080p
  - macOS: 内部用 CGImage，~20ms@1080p
  - Linux: 内部用 XGetImage，~40ms@1080p

优点：一行代码搞定跨平台截图，适合快速验证整个数据流
缺点：性能不如原生 API，不支持窗口截图、多显示器精确枚举
```

### 3.2 最小实现：ScreenshotService + Qt 截图

**新建文件：** `src/services/screenshot/ScreenshotTypes.h`

```cpp
#pragma once
#include <QImage>
#include <QRect>
#include <QDateTime>
#include <QString>
#include <cstdint>

using ScreenshotId = uint64_t;

enum class CaptureMode : uint8_t {
    FullScreen = 0,
    Region     = 1,
    Window     = 2,
    Delayed    = 3,
};

enum class ExportFormat : uint8_t {
    PNG  = 0,
    JPEG = 1,
    WebP = 2,
};

enum class FinalizeAction : uint8_t {
    SaveToFile      = 0,
    CopyToClipboard = 1,
    SaveAndCopy     = 2,
    Discard         = 3,
};

enum class ScreenshotState : uint8_t {
    Idle       = 0,
    Pending    = 1,   // 等待用户选区
    Captured   = 2,   // 已捕获
    Annotating = 3,   // 标注中
    Saving     = 4,
};

struct CaptureRequest {
    CaptureMode mode = CaptureMode::FullScreen;
    int         screenIndex = 0;
    QRect       region;            // Region 模式
    int         delayMs = 0;       // Delayed 模式
    bool        showCursor = false;
};

struct Screenshot {
    ScreenshotId id{0};
    QImage       rawImage;         // 物理像素
    QRect        capturedRegion;
    double       dpr{1.0};
    QDateTime    capturedAt;
    CaptureMode  mode{CaptureMode::FullScreen};
    QString      savedPath;

    [[nodiscard]] QSize logicalSize() const noexcept {
        return (rawImage.size() / dpr).toSize();
    }
};
```

```cpp
private:
    void transitionTo(ScreenshotState newState);

    std::shared_ptr<Screenshot>  m_current;
```

**新建文件：** `src/services/screenshot/ScreenshotService.cpp`

```cpp
#include "services/screenshot/ScreenshotService.h"

#include <QGuiApplication>
#include <QScreen>
#include <QClipboard>
#include <QFile>
#include <QDir>
#include <QElapsedTimer>

#include "core/log/Logger.h"
#include "core/log/LogModules.h"

ScreenshotService::ScreenshotService(QObject* parent)
    : QObject(parent) {
    TB_LOG_INFO(LogModule::Screenshot, "ScreenshotService created (Qt native mode)");
}

ScreenshotState ScreenshotService::state() const noexcept { return m_state; }
std::shared_ptr<Screenshot> ScreenshotService::currentShot() const noexcept { return m_current; }

void ScreenshotService::transitionTo(ScreenshotState newState) {
    if (m_state != newState) {
        m_state = newState;
        emit stateChanged(m_state);
        TB_LOG_DEBUG(LogModule::Screenshot,
            "State transitioned to {}", static_cast<int>(m_state));
    }
}

void ScreenshotService::captureFullScreen(int screenIndex, CaptureCallback cb) {
    transitionTo(ScreenshotState::Pending);

    const auto screens = QGuiApplication::screens();
    if (screenIndex < 0 || screenIndex >= screens.size()) {
        const QString err = QStringLiteral("Invalid screen index: %1").arg(screenIndex);
        transitionTo(ScreenshotState::Idle);
        emit captureError(err);
        if (cb) cb(Result<std::shared_ptr<Screenshot>>::err(
            TB_MAKE_ERROR(ErrorCode::ScreenCaptureError, err.toStdString())));
        return;
    }

    QScreen* screen = screens[screenIndex];
    QElapsedTimer timer;
    timer.start();

    // 关键一行：Qt 原生截图
    QPixmap pixmap = screen->grabWindow(0);

    const int elapsed = static_cast<int>(timer.elapsed());
    TB_LOG_INFO(LogModule::Screenshot,
        "Qt grabWindow done | screen='{}' size={}x{} elapsed={}ms",
        screen->name().toStdString(),
        pixmap.width(), pixmap.height(), elapsed);

    if (pixmap.isNull()) {
        const QString err = QStringLiteral("grabWindow returned null pixmap");
        transitionTo(ScreenshotState::Idle);
        emit captureError(err);
        if (cb) cb(Result<std::shared_ptr<Screenshot>>::err(
            TB_MAKE_ERROR(ErrorCode::ScreenCaptureError, err.toStdString())));
        return;
    }

    auto shot = std::make_shared<Screenshot>();
    shot->id = m_nextId++;
    shot->rawImage = pixmap.toImage().convertToFormat(QImage::Format_ARGB32_Premultiplied);
    shot->capturedRegion = screen->geometry();
    shot->dpr = screen->devicePixelRatio();
    shot->capturedAt = QDateTime::currentDateTime();
    shot->mode = CaptureMode::FullScreen;

    m_current = shot;
    transitionTo(ScreenshotState::Captured);
    emit captureReady(shot);

    if (cb) cb(Result<std::shared_ptr<Screenshot>>::ok(shot));
}

void ScreenshotService::captureRegion(const QRect& region, int screenIndex, CaptureCallback cb) {
    // 先截全屏，再裁剪
    captureFullScreen(screenIndex, [this, region, cb](Result<std::shared_ptr<Screenshot>> result) {
        if (result.isErr()) {
            if (cb) cb(std::move(result));
            return;
        }

        auto shot = std::move(result).value();
        const QRect physicalRegion(
            static_cast<int>(region.x() * shot->dpr),
            static_cast<int>(region.y() * shot->dpr),
            static_cast<int>(region.width() * shot->dpr),
            static_cast<int>(region.height() * shot->dpr));

        // 确保裁剪区域在图像范围内
        const QRect validRegion = physicalRegion.intersected(shot->rawImage.rect());
        if (validRegion.isEmpty()) {
            if (cb) cb(Result<std::shared_ptr<Screenshot>>::err(
                TB_MAKE_ERROR(ErrorCode::InvalidArgument, "Region is outside screen bounds")));
            return;
        }

        shot->rawImage = shot->rawImage.copy(validRegion);
        shot->capturedRegion = region;
        shot->mode = CaptureMode::Region;

        TB_LOG_INFO(LogModule::Screenshot,
            "Region captured | region={}x{} physical={}x{}",
            region.width(), region.height(),
            validRegion.width(), validRegion.height());

        if (cb) cb(Result<std::shared_ptr<Screenshot>>::ok(shot));
    });
}

Result<QString> ScreenshotService::saveToFile(const QString& savePath, ExportFormat format) {
    if (!m_current || m_current->rawImage.isNull()) {
        return Result<QString>::err(
            TB_MAKE_ERROR(ErrorCode::InvalidArgument, "No screenshot to save"));
    }

    // 确保目录存在
    const QFileInfo fileInfo(savePath);
    QDir().mkpath(fileInfo.absolutePath());

    // 选择格式
    const char* fmt = "PNG";
    int quality = -1;
    switch (format) {
        case ExportFormat::PNG:  fmt = "PNG";  break;
        case ExportFormat::JPEG: fmt = "JPEG"; quality = 90; break;
        case ExportFormat::WebP: fmt = "WEBP"; quality = 90; break;
    }

    QElapsedTimer timer;
    timer.start();

    if (!m_current->rawImage.save(savePath, fmt, quality)) {
        return Result<QString>::err(
            TB_MAKE_ERROR(ErrorCode::IoError,
                          "Failed to save screenshot to: " + savePath.toStdString()));
    }

    const int elapsed = static_cast<int>(timer.elapsed());
    m_current->savedPath = savePath;

    TB_LOG_INFO(LogModule::Screenshot,
        "Screenshot saved | path='{}' format={} elapsed={}ms",
        savePath.toStdString(), fmt, elapsed);

    emit savedToFile(savePath);
    return Result<QString>::ok(savePath);
}

Result<void> ScreenshotService::copyToClipboard() {
    if (!m_current || m_current->rawImage.isNull()) {
        return Result<void>::err(
            TB_MAKE_ERROR(ErrorCode::InvalidArgument, "No screenshot to copy"));
    }

    QClipboard* clipboard = QGuiApplication::clipboard();
    if (!clipboard) {
        return Result<void>::err(
            TB_MAKE_ERROR(ErrorCode::IoError, "Cannot access system clipboard"));
    }

    clipboard->setImage(m_current->rawImage);
    m_current->savedPath.clear();  // 标记为"仅复制"

    TB_LOG_INFO(LogModule::Screenshot, "Screenshot copied to clipboard");
    emit copiedToClipboard();
    return Result<void>::ok();
}

void ScreenshotService::cancel() {
    m_current.reset();
    transitionTo(ScreenshotState::Idle);
    TB_LOG_DEBUG(LogModule::Screenshot, "Screenshot cancelled");
}
```

## 4. Phase 2：全屏覆盖层 + 区域选择

### 4.1 核心难点

区域选择的关键是一个**全屏透明覆盖窗口**：

```
┌─────────────────────────────────────────────────────────────┐
│  ScreenshotOverlay (Window)                                  │
│  flags: FramelessWindowHint | WindowStaysOnTopHint | Tool    │
│  visibility: FullScreen                                      │
│  color: transparent                                          │
│                                                              │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │  背景层：全屏截图 Image（半透明暗化遮罩）                  │ │
│  │                                                          │ │
│  │    ┌──────────────┐                                      │ │
│  │    │  选区（亮色） │  ← 用户拖拽产生的矩形                 │ │
│  │    │  1920 × 1080 │                                      │ │
│  │    └──────────────┘                                      │ │
│  │                                                          │ │
│  │  放大镜（跟随鼠标）         工具栏（选区确认后出现）        │ │
│  └─────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 QML Overlay 实现要点

```qml
// ScreenshotOverlay.qml — 关键结构
Window {
    id: overlay

    // ① 窗口必须无边框、全屏、置顶
    flags: Qt.FramelessWindowHint
         | Qt.WindowStaysOnTopHint
         | Qt.Tool
         | Qt.BypassWindowManagerHint   // Linux 必须
    visibility: Window.FullScreen
    color: "transparent"

    // ② 背景：显示截图 + 暗化遮罩
    Image {
        id: backgroundImage
        anchors.fill: parent
        source: "image://screenshot/bg"   // C++ ImageProvider
        cache: false
    }

    // ③ 选区交互
    MouseArea {
        anchors.fill: parent
        hoverEnabled: true
        cursorShape: Qt.CrossCursor

        property point startPos
        property bool selecting: false

        onPressed: (m) => {
            startPos = Qt.point(m.x, m.y)
            selecting = true
            selectionRect.x = m.x
            selectionRect.y = m.y
            selectionRect.width = 0
            selectionRect.height = 0
        }

        onPositionChanged: (m) => {
            if (selecting) {
                // 实时更新选区矩形
                selectionRect.x = Math.min(startPos.x, m.x)
                selectionRect.y = Math.min(startPos.y, m.y)
                selectionRect.width = Math.abs(m.x - startPos.x)
                selectionRect.height = Math.abs(m.y - startPos.y)
            }
        }

        onReleased: (m) => {
            selecting = false
            if (selectionRect.width > 5 && selectionRect.height > 5) {
                // 选区确认 → 通知 ViewModel
                viewModel.selectionConfirmed(Qt.rect(
                    selectionRect.x, selectionRect.y,
                    selectionRect.width, selectionRect.height))
            }
        }
    }

    // ④ 选区矩形显示
    Rectangle {
        id: selectionRect
        color: "transparent"
        border.color: "#00BFFF"
        border.width: 2
        visible: width > 0 && height > 0
    }

    // ⑤ 暗化遮罩（选区外变暗，选区内透明）
    // 用 4 个矩形拼接实现"挖洞"效果
    // 上方遮罩
    Rectangle {
        x: 0; y: 0
        width: parent.width
        height: selectionRect.y
        color: "#80000000"
    }
    // 下方遮罩
    Rectangle {
        x: 0; y: selectionRect.y + selectionRect.height
        width: parent.width
        height: parent.height - y
        color: "#80000000"
    }
    // 左侧遮罩
    Rectangle {
        x: 0; y: selectionRect.y
        width: selectionRect.x
        height: selectionRect.height
        color: "#80000000"
    }
    // 右侧遮罩
    Rectangle {
        x: selectionRect.x + selectionRect.width; y: selectionRect.y
        width: parent.width - x
        height: selectionRect.height
        color: "#80000000"
    }

    // ⑥ 选区尺寸提示
    Text {
        x: selectionRect.x
        y: selectionRect.y - 24
        text: selectionRect.width + " × " + selectionRect.height
        color: "white"
        font.pixelSize: 12
        style: Text.Outline; styleColor: "black"
    }

    // ⑦ 键盘快捷键
    Keys.onEscapePressed: overlay.close()
    Keys.onReturnPressed: { /* 确认截图 */ }
}
```

## 5. Phase 3：标注引擎

### 5.1 核心架构：Command Pattern

```
用户操作              命令对象              CommandHistory         AnnotationLayer
─────────            ──────────           ──────────────         ───────────────
画矩形    →    AddShapeCommand    →    push(cmd)    →    layer.shapes.push_back(shape)
                     │
                  undo()               undo()           layer.shapes.pop_back()
                     │
                  redo()               redo()           layer.shapes.push_back(shape)
```

**关键设计原则：所有对 AnnotationLayer 的修改都必须通过 Command 执行。**

### 5.2 实现顺序

```
1. AnnotationTypes.h     ← 数据结构（RectShape, ArrowShape, ...）
2. IAnnotationCommand    ← 命令接口
3. CommandHistory        ← 撤销/重做栈
4. AddShapeCommand       ← 第一个具体命令
5. ShapeRenderer         ← 渲染器（QPainter 绘制各形状）
6. AnnotationEngine      ← 引擎（组合以上所有）
7. AnnotationCanvas.qml  ← QML 画布（Canvas 2D + MouseArea）
```

### 5.3 QML Canvas 与 C++ 的交互

```qml
// AnnotationCanvas.qml
Canvas {
    id: canvas
    renderTarget: Canvas.FramebufferObject   // GPU 渲染，60fps

    onPaint: {
        var ctx = getContext("2d")
        ctx.clearRect(0, 0, width, height)
        // 通知 C++ 渲染所有标注到此 Canvas
        // 方法 1：C++ 注册一个 Q_INVOKABLE 方法
        //   ScreenshotViewModel.renderToCanvas(ctx)
        // 方法 2：C++ 提供一个 QImage，QML 绘制它
        //   ctx.drawImage(screenshotViewModel.annotationImage, 0, 0)
    }

    // 监听 C++ 信号刷新
    Connections {
        target: screenshotViewModel
        function onAnnotationLayerNeedsRepaint() {
            canvas.requestPaint()
        }
    }
}

// 鼠标事件转发给 C++
MouseArea {
    anchors.fill: canvas
    onPressed:      (m) => screenshotViewModel.onMousePress(m.x, m.y)
    onPositionChanged: (m) => screenshotViewModel.onMouseMove(m.x, m.y)
    onReleased:     (m) => screenshotViewModel.onMouseRelease(m.x, m.y)
}
```

### 5.4 标注渲染的技术选择

```
方案 A：QML Canvas 2D（推荐 Phase 3）
  - 用 QQuickItem 的 updatePaintNode() 或 Canvas.onPaint
  - C++ 端用 QPainter 绘制到 QImage，传给 QML
  - 简单直接，适合 MVP

方案 C：QSGNode（场景图节点）
  - 最高性能，直接操作 Qt 场景图
  - 实现复杂，适合 Phase 5 极致优化
```

## 6. Phase 4：平台原生截图优化

### 6.1 为什么要替换 Qt 原生截图

```
QScreen::grabWindow() 的问题：
  - Windows: ~30ms（BitBlt），不支持 GPU 加速
  - macOS: ~20ms，但无法捕获 GPU 渲染的窗口内容
  - Linux: ~40ms（XGetImage），Wayland 下完全不工作

原生 API 的优势：
  - Windows DXGI: ~8ms（GPU 直接读取显存）
  - macOS CGImage: ~10ms（系统级优化）
  - Linux XShm: ~15ms（共享内存零拷贝）
```

### 6.2 工厂模式切换

```cpp
// IScreenCapture.h — 抽象接口

public:
    virtual ~IScreenCapture() = default;
    [[nodiscard]] virtual Result<QImage> captureFullScreen(int screenIndex) = 0;
    [[nodiscard]] virtual Result<QImage> captureRegion(const QRect& region) = 0;
    [[nodiscard]] virtual Result<QImage> captureWindow(WId windowId) = 0;
};

class ScreenCaptureFactory {
public:
    static std::unique_ptr<IScreenCapture> create() {
#if defined(Q_OS_WIN)
        return std::make_unique<Win32ScreenCapture>();
#elif defined(Q_OS_MAC)
        return std::make_unique<MacScreenCapture>();
#elif defined(Q_OS_LINUX)

#endif
    }

// ScreenshotService 构造函数中注入
```

### 6.3 Windows DXGI 实现要点

```
DXGI Desktop Duplication 工作流程：
  1. D3D11CreateDevice() 创建 D3D 设备
  2. IDXGIOutput1::DuplicateOutput() 获取桌面副本
  3. IDXGIOutputDuplication::AcquireNextFrame() 获取帧
  4. ID3D11Texture2D → Map() → 直接得到像素数据
  5. 构建 QImage（零拷贝或一次 memcpy）

性能：~8ms@1080p，~15ms@4K

回退策略：
  - DXGI 不可用（Win7、远程桌面）→ BitBlt（GDI）
  - DXGI AccessLost（GPU 重置）→ 重建设备
```

### 6.4 macOS 权限处理

```
macOS 10.15+ 需要"屏幕录制"权限。

检测方式：
  CGDisplayStreamCreate() 返回 nil → 无权限

引导用户：
  "请在 系统设置 → 隐私与安全性 → 屏幕录制 中授权 IQtools Plus"

代码签名：
  正式发布的 app 必须签名，否则 macOS 不会弹出权限请求
```

### 6.5 Linux Wayland 兼容

```
Wayland 下 X11 API 完全不工作（安全设计）。

方案 1（推荐）：使用 xdg-desktop-portal
  - D-Bus 接口：org.freedesktop.portal.Screenshot
  - 系统弹出权限对话框 → 用户确认 → 返回截图
  - 兼容所有主流桌面环境（GNOME、KDE、Sway）

方案 2：wlr-screencopy 协议
  - 仅支持 wlroots 合成器（Sway、Hyprland）
  - 更底层，无权限对话框

方案 3（回退）：QScreen::grabWindow()
  - 某些合成器下只能截取本进程窗口
  - 功能受限但至少不崩溃
```

## 8. 跨平台核心难题详解

### 8.1 HiDPI / Retina 缩放

```
问题：逻辑坐标 ≠ 物理像素

  macOS Retina: 逻辑 1440×900 → 物理 2880×1800（DPR=2.0）
  Windows 150%: 逻辑 1280×720 → 物理 1920×1080（DPR=1.5）

QML 中的坐标是逻辑坐标。
截图 API 返回的是物理像素。

必须做坐标转换：
  physicalX = logicalX × dpr
  physicalY = logicalY × dpr

Qt 的 QScreen::devicePixelRatio() 返回 DPR。
```

### 8.2 多显示器

```
  显示器 0: 1920×1080 at (0, 0)
  显示器 1: 2560×1440 at (1920, 0)

用户在显示器 1 上框选区域 (100, 100, 500, 300)
  → 逻辑坐标：(2020, 100, 500, 300)   // 加上显示器偏移
  → 物理坐标：(2020×dpr, 100×dpr, 500×dpr, 300×dpr)

截图时需要知道选区落在哪个显示器上，然后用该显示器的 DPR。
```

### 8.3 截图时隐藏自身窗口

```
问题：截图时如果主窗口可见，截图结果中会包含主窗口。

解决方案：
  1. 截图前隐藏主窗口（hide()）
  2. 等待一帧（QTimer::singleShot(0, ...)）
  3. 执行截图
  4. 截图完成后恢复主窗口（show()）

  // 但全屏覆盖层不需要这个——因为覆盖层本身是透明的。
```

### 8.4 全屏覆盖层在 Linux 下的兼容性

```qml

// 回退方案：使用 normal window + 最大化 + 置顶
flags: Qt.FramelessWindowHint | Qt.WindowStaysOnTopHint

//   visibility: Window.FullScreen
//   // 并在 Component.onCompleted 中手动设置 geometry 为屏幕尺寸
```

## 9. 关键设计决策

### 9.1 截图线程模型

```
  主线程（GUI）                    截图线程
  ──────────────                   ────────────
  用户按热键                        │
       │                           │
       ▼                           │
  ViewModel::startCapture()        │
       │                           │
       ├──────────────────────────►│
       │                    captureScreen()
       │                    (DXGI/CGImage/XShm)
       │                    ~10-30ms
       │                           │
       │◄─────QueuedConnection─────┤
       │                    captureReady(image)
       ▼                           │
  显示 Overlay                      │
```

**关键：截图 API 在专用线程中执行，通过 QueuedConnection 将结果传回主线程。**

Phase 1 可以先在主线程执行（QScreen::grabWindow 只需 ~30ms，不会明显卡顿）。
Phase 4 必须移到专用线程。

### 9.2 标注数据存不存储

```
标注数据（AnnotationLayer）只存在于内存中，不持久化。

理由：
  - 标注是"最后一步"操作，完成后就 flatten() 合并到图像中
  - 持久化标注数据需要序列化所有 Shape variant，复杂度高
  - 用户期望保存的是一张"带标注的图片"，不是可编辑的标注数据

历史中的截图：
  - rawImage 保留（可重新编辑标注）
  - 如果用户从历史中重新打开，创建新的 AnnotationLayer
```

### 9.3 ImageProvider vs QImage 直传

```
QML 显示 C++ QImage 的两种方式：

方式 1：QQuickImageProvider（推荐用于大图）
  - C++ 注册一个 image provider
  - QML 用 source: "image://provider/name" 异步加载
  - 支持缓存、异步解码
  - 适合：全屏背景图、历史缩略图

方式 2：Q_PROPERTY<QImage>（推荐用于频繁更新）
  - ViewModel 暴露 QImage 属性
  - QML Image { source: viewModel.screenshotImage }
  - 同步加载，适合小图或频繁刷新
  - 注意：每次属性变化都会触发 QML 重新加载
```

```cpp
Q_INVOKABLE QColor pickColor(int x, int y) const {
    if (!m_currentShot || m_currentShot->rawImage.isNull())
        return QColor();

    const int px = static_cast<int>(x * m_currentShot->dpr);
    const int py = static_cast<int>(y * m_currentShot->dpr);

    if (px < 0 || py < 0 ||
        px >= m_currentShot->rawImage.width() ||
        py >= m_currentShot->rawImage.height())

}
```

### 11.3 放大镜（Magnifier）完整实现

微信截图的放大镜是核心体验——一个跟随鼠标的圆形窗口，显示鼠标周围像素的放大视图，中心有十字线，底部有色值和坐标。

```qml
// ── 放大镜组件 ────────────────────────────────────────────────────────
Item {
    id: magnifier

    property alias sourceImage: magnifiedContent.source   // 全屏截图
    property real  zoomFactor: 8          // 放大倍数
    property int   radius: 80            // 放大镜半径（像素）
    property real  mouseX: 0
    property real  mouseY: 0
    property color currentColor: "transparent"

    width:  radius * 2
    height: radius * 2

    // 跟随鼠标，偏移避免遮挡光标和选区
    // 优先显示在鼠标右下方，空间不足时切换到其他方向
    x: {
        var preferred = mouseX + 20
        if (preferred + width > parent.width - 4)
            return mouseX - width - 20   // 右侧空间不足，放左侧
        return preferred
    }
    y: {
        var preferred = mouseY + 20
        if (preferred + height + 40 > parent.height)   // 40 = 色值标签高度
            return mouseY - height - 20  // 下方空间不足，放上方
        return preferred
    }

    // ── 圆形裁切容器 ──────────────────────────────────────────────────
    Item {
        id: clipContainer
        anchors.fill: parent
        layer.enabled: true
        layer.effect: ShaderEffect {
            // 圆形裁切 shader：超出圆的部分透明
            fragmentShader: "
                varying highp vec2 qt_TexCoord0;
                uniform sampler2D source;
                uniform highp float radius;
                void main() {
                    highp vec2 uv = qt_TexCoord0 - vec2(0.5);
                    highp float dist = length(uv);
                    if (dist > radius) discard;
                    gl_FragColor = texture2D(source, qt_TexCoord0);
                }
            "
        }

        // 放大后的内容
        Image {
            id: magnifiedContent
            // 关键：sourceRect 从全屏截图中取鼠标周围的区域
            // 然后放大显示
            width: magnifier.width
            height: magnifier.height
            fillMode: Image.Stretch

            // 计算源区域：以鼠标为中心，半径 = magnifierRadius / zoomFactor
            sourceRect {
                x: magnifier.mouseX * dpr - (magnifier.radius / magnifier.zoomFactor)
                y: magnifier.mouseY * dpr - (magnifier.radius / magnifier.zoomFactor)
                width:  (magnifier.radius * 2) / magnifier.zoomFactor
                height: (magnifier.radius * 2) / magnifier.zoomFactor
            }
            // 注意：sourceRect 使用物理像素坐标
            // dpr 来自 ScreenshotViewModel
        }

        // 网格线（像素边界）
        Canvas {
            anchors.fill: parent
            visible: magnifier.zoomFactor >= 6   // 倍率够大时显示网格

            onPaint: {
                var ctx = getContext("2d")
```

### 11.5 选区 8 控制点（拖拽调整大小）

选区确认后，四角和四边中点各有一个小方块，拖拽可调整大小：

```qml
// ── 选区控制点 ────────────────────────────────────────────────────────
Item {
    id: resizeHandles
    visible: selectionLayer.hasSelection && !annotationMode

    // 8 个控制点的位置定义
    // nw=左上, n=上, ne=右上, e=右, se=右下, s=下, sw=左下, w=左
    Repeater {
        model: [
            { name: "nw", x: 0,            y: 0,             cursor: Qt.SizeFDiagCursor },
            { name: "n",  x: 0.5,          y: 0,             cursor: Qt.SizeVerCursor   },
            { name: "ne", x: 1,            y: 0,             cursor: Qt.SizeBDiagCursor },
            { name: "e",  x: 1,            y: 0.5,           cursor: Qt.SizeHorCursor   },
            { name: "se", x: 1,            y: 1,             cursor: Qt.SizeFDiagCursor },
            { name: "s",  x: 0.5,          y: 1,             cursor: Qt.SizeVerCursor   },
            { name: "sw", x: 0,            y: 1,             cursor: Qt.SizeBDiagCursor },
            { name: "w",  x: 0,            y: 0.5,           cursor: Qt.SizeHorCursor   },
        ]

        delegate: Rectangle {
            required property var modelData
            required property int index

            x: selectionRect.x + selectionRect.width  * modelData.x - 4
            y: selectionRect.y + selectionRect.height * modelData.y - 4
            width: 8; height: 8
            radius: 1
            color: "#FFFFFF"
            border.color: "#00BFFF"
            border.width: 1

            MouseArea {
                anchors.fill: parent
                anchors.margins: -4   // 扩大触摸区域
                cursorShape: modelData.cursor
                property point startPos
                property rect startRect

                onPressed: {
                    startPos = Qt.point(mouseX, mouseY)
                    startRect = Qt.rect(
                        selectionRect.x, selectionRect.y,
                        selectionRect.width, selectionRect.height)
                }

                onPositionChanged: {
                    if (!pressed) return
                    var dx = mouseX - startPos.x
                    var dy = mouseY - startPos.y
                    // 根据控制点名称计算新的选区矩形
                    resizeSelection(modelData.name, startRect, dx, dy)
                }
            }
        }
    }

    function resizeSelection(handle, startRect, dx, dy) {
        var r = Qt.rect(startRect.x, startRect.y, startRect.width, startRect.height)

        switch (handle) {
        case "nw": r.x += dx; r.y += dy; r.width -= dx; r.height -= dy; break
        case "n":  r.y += dy; r.height -= dy; break
        case "ne": r.y += dy; r.width += dx; r.height -= dy; break
        case "e":  r.width += dx; break
        case "se": r.width += dx; r.height += dy; break
        case "s":  r.height += dy; break
        case "sw": r.x += dx; r.width -= dx; r.height += dy; break
        case "w":  r.x += dx; r.width -= dx; break
        }

        // 最小尺寸限制
        if (r.width >= 10 && r.height >= 10) {
            selectionRect = r
        }
    }
}
```

### 11.13 性能关键点

```
1. 放大镜 sourceRect 刷新频率
   - 使用 Image.sourceRect 而非手动裁剪 QImage
   - QML Image 的 sourceRect 变化时只重新解码对应区域
   - 但如果放大镜每帧都刷新（鼠标移动），需确保 sourceRect 变化是平滑的
   - 技巧：用 Behavior on sourceRect { enabled: false } 禁用动画

2. 暗化遮罩的 4 块矩形
   - 比用 Canvas 绘制高效得多
   - 选区变化时只需更新 4 个 Rectangle 的 x/y/width/height
   - 不需要 requestPaint()

3. 标注 Canvas 的刷新策略
   - 仅在标注数据变化时 requestPaint()
   - 使用 Canvas.FramebufferObject（GPU 渲染）
   - 自由笔绘制时，每 2-3 个像素采样一次，避免过多 shape 点

4. 放大镜网格线
   - 仅在 zoomFactor >= 6 时绘制
   - 用 Canvas 绘制，缓存结果，仅在 zoomFactor 变化时重绘
```
