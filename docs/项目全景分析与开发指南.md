## 一、项目概览

**IQtools Plus** 是一款基于 **C++20 / Qt 6.6+ / QML** 的桌面效率工具箱应用，采用 MVVM 分层架构，目前处于 **功能实现阶段**（文档体系已完成，核心功能模块正在搭建）。

### 1.1 核心技术栈

| 层面 | 技术 |
|------|------|
| 语言 | C++20（后端）/ QML（前端） |
| 框架 | Qt 6.6+（Quick, QuickControls2, Network, Sql） |
| 构建 | CMake 3.25+ |
| 日志 | spdlog 1.12+ |
| 测试 | Google Test + Qt Test |
| 错误处理 | 自研 `Result<T>` 类型（替代异常） |

### 1.2 功能模块一览

| 模块 | 说明 | 当前状态 |
|------|------|----------|
| **首页 (Home)** | 仪表盘，模块导航 | ✅ 已实现 |
| **AI 助手** | 对话式 AI 交互 | ✅ 接口+Mock+ViewModel+UI 完成 |
| **翻译** | 多引擎文本翻译 | ✅ 接口+Mock+ViewModel+UI 完成 |
| **剪贴板** | 历史管理、搜索、分类 | ⬜ 仅 UI 骨架，Service 层待实现 |

## 二、架构设计详解

### 2.1 分层架构图

```
┌─────────────────────────────────────────────────────┐
│              QML UI Layer (src/ui/)                 │
│   main.qml / pages/*.qml / components/*.qml        │
│   纯声明式界面，不含业务逻辑                           │
└──────────────────────┬──────────────────────────────┘
                       │  Q_PROPERTY / Q_INVOKABLE / signals
┌──────────────────────▼──────────────────────────────┐
│           ViewModel Layer (src/viewmodels/)          │
│   AppShellViewModel / TranslateViewModel / ...      │
│   QObject 子类，桥接 C++ ↔ QML，管理 UI 状态         │
└──────────────────────┬──────────────────────────────┘
                       │  C++ 函数调用 / std::function 回调
┌──────────────────────▼──────────────────────────────┐
│            Service Layer (src/services/)             │
│   纯业务逻辑，通过接口（纯虚类）定义，不依赖 Qt UI      │
│   translate/ | ai/ | screenshot/ | clipboard/ | hotkey/
└──────────────────────┬──────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────┐
│          Core Infrastructure (src/core/)             │
│   Error (Result<T>) | Logger | EventBus | Config    │
│   Plugin | Thread                                   │
└─────────────────────────────────────────────────────┘
```

### 2.2 依赖注入机制

项目使用 `AppContext` 作为组合根（Composition Root），在 `main.cpp` 中注册所有服务：

```cpp
// main.cpp
AppContext appContext;
appContext.registerService<ITranslateService>(std::make_shared<MockTranslateService>());
appContext.registerService<IAIService>(std::make_shared<MockAIService>());

AppShellViewModel appShellViewModel;
appShellViewModel.setAppContext(&appContext);  // 注入到 ViewModel 树
```

`AppShellViewModel` 在 `setAppContext()` 中将服务分发给各子 ViewModel：

```
AppShellViewModel
  ├── HomeViewModel
  ├── TranslateViewModel     ← injectService(ITranslateService)
  ├── AIAssistantViewModel   ← injectService(IAIService)
  ├── LogViewModel
  └── SettingViewModel
```

### 2.3 关键设计模式

| 模式 | 应用位置 | 说明 |
|------|----------|------|
| **MVVM** | 全局 | QML ↔ ViewModel ↔ Service |
| **依赖注入** | AppContext | 服务生命周期管理，便于测试/替换 |
| **接口隔离** | ITranslateService, IAIService | 纯虚接口，实现可替换 |
| **Result\<T\>** | Error.h | 替代异常的错误传播机制 |
| **信号/槽** | ViewModel ↔ QML | Qt 原生的观察者模式 |

### 4.1 翻译请求完整数据流

```
用户在 QML 输入文本，点击"翻译"按钮

  │  viewModel.translate()

  ▼

  │  → QTimer::singleShot(280ms, ...) 模拟延迟

  │  → callback(Result<TranslateResult>::ok(result))
  ▼
TranslateViewModel 回调处理
  │  → setResultText(result.translatedText)
  │  → emit resultTextChanged() / translateSucceeded()
  ▼
TranslatePage.qml 响应信号
  │  → 更新结果文本框
  │  → 显示翻译耗时
```

## 五、错误处理规范

项目使用 `Result<T>` 替代 C++ 异常进行错误传播。**所有 Service 层返回值必须使用 `Result<T>`**。

```cpp
// Error.h 核心类型
enum class ErrorCode : int32_t {
    Ok = 0, Unknown = 1, InvalidArgument = 2, Timeout = 3,
    NetworkError = 4, PermissionDenied = 5, ...
};

// 成功返回
auto result = Result<int>::ok(42);

// 失败返回
auto result = Result<int>::err(
    TB_MAKE_ERROR(ErrorCode::InvalidArgument, "Text cannot be empty"));

// 错误检查
if (result.isErr()) {
    TB_LOG_ERROR(LogModule::Translate, "{}", result.error().toString());
    return;
}

// 获取值（仅在 isOk() 后调用）
int value = std::move(result).value();

// 链式错误传播
TB_RETURN_IF_ERR(someOperation());
```

### 8.3 添加新模块的通用步骤

以添加一个新 Service 为例：

**Step 1：定义接口**
```cpp

public:
    using Ptr = std::shared_ptr<IXxxService>;
```

**Step 2：实现 Mock**
```cpp
// src/services/xxx/MockXxxService.h/.cpp
class MockXxxService : public QObject, public IXxxService { ... };
```

**Step 3：创建 ViewModel**
```cpp
// src/viewmodels/XxxViewModel.h/.cpp
class XxxViewModel final : public QObject {

    // Q_PROPERTY, Q_INVOKABLE, injectService() ...
```

**Step 4：创建 QML 页面**
```qml
// src/ui/pages/XxxPage.qml
```

## 九、关键注意事项

1. **QML 不直接访问 Service 层** — 必须通过 ViewModel 中转
2. **服务注入时机** — `setAppContext()` 在 `main.cpp` 中 QML 引擎加载前调用
3. **跨线程通信** — 非 GUI 线程更新 UI 必须用 `QMetaObject::invokeMethod(..., Qt::QueuedConnection)`
4. **错误传播** — Service 层用 `Result<T>`，ViewModel 层转换为 Qt 信号
5. **头文件顺序** — 本项目头文件 → Qt 头文件 → 第三方库 → 标准库
6. **QML_ELEMENT** — ViewModel 使用此宏自动注册，无需手动 `qmlRegisterType`
7. **主题系统** — QML 中通过 `Theme.qml` 单例读取，不要硬编码颜色值
8. **文档先行** — 每个模块都有对应的设计文档（`docs/` 目录），开发前先阅读
